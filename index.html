// The four tools

📊

ctxguard

ulimit for Claude Code's 200k context window. warn | compress | kill when your agent burns too many tokens.

cargo install ctxguard
🔍

recall-ai

Search every AI conversation you've ever had, locally. SQLite + FTS5 over ChatGPT / Claude / Claude Code / Codex exports.

cargo install recall-ai
📌

pinpoint

Your ~/.bash_history but for AI prompts. Save the keepers with one keystroke (pp shell helper).

cargo install pinpoint
🛡️

mcp-sentry

Policy-as-code firewall for MCP servers. Approve / deny / audit every tools/call from your AI agent.

cargo install mcp-sentry

// Why this kit exists

The AI coding era is leaky. We plug the holes.

Without these tools

  • You write 50 prompts/day, lose the keepers by Friday.
  • Your agent re-reads auth.ts 30 times. Wallet doesn't scream (prompt caching hides it), but context window does.
  • That MCP server you installed — is it reading ~/.ssh? You don't know.
  • "What was that bug fix from 3 weeks ago?" → grep, scroll, give up.

With this kit

  • pp "fix the auth bug" → saved with auto-name + tags.
  • Agent hits 80k tokens → ctxguard warns or compresses via /compact.
  • Every tools/call goes through mcp-sentry policy.
  • recall-ai find "auth bug" → FTS5 in 20 ms.

Built solo. MIT OR Apache-2.0. 100% local.

Five days, four production CLIs, one coherent kit. Same author, same stack, same release cadence. No SaaS, no telemetry, no lock-in. Just five single binaries that do their job.
